Stop Abusing Configuration Management

Configuration Management frameworks such as Puppet, Chef, CfEngine, etc. are incredibly powerful. In fact, they are too powerful. Systems Engineers use them too often use them to duct tape together overly complex architectures when there are often better techniques to accomplish the same goal that will result in simpler, more reliable applications. * Write proper f*ing daemons * Get a proper runtime * When to move configuration up the application stack w/ tools like [archaius](https://github.com/Netflix/archaius) * Apache reloading sucks, get thee a dynamic http proxy * Pre-bake your binary dependencies when possible onto your boot images* and more! * Time available, I hope to discuss how using ZooKeeper can simplify configuration management tasks.
